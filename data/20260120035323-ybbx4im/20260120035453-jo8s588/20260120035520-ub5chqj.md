# 笔记系统--思源管理

# 内容架构：【两文件+一文件夹】

## 一、优化日志--表格形式

   包含四个部分：

1. 【问题/灵感--对具体知识】

    - 问题：发现哪里不好
    - 灵感：发现哪里可优化

    1. 自己想出来的
    2. 从别处触类来的

        触类的来源可近可远----例如写文，可以是同平台文、可以是其它短篇、可以是长篇网文、可以是影视......
2. 【问题/灵感--对系统架构】

    分类同上
3. 【更新哪些内容】

    新增/废弃/修改/合并等
4. （可选）【产出链接】

‍

样式：

|问题/灵感--对具体知识|问题/灵感--对系统架构|更新哪些内容|产出链接|
| -----------------------| -----------------------| --------------| ----------|
|||||

‍

## 二、内容地图

【笔记链接及组合+整体说明】

汇集了所有笔记链接、并进行系统化组合，并进行大面上的简要说明

- 为何是地图、而非树状：笔记链接不可能全部完美归类成树状目录，所以需要灵活组合

‍

## 三、知识库

放各种研究软件

# 思源的使用要点

### 所有配置迁移

将这几部分弄完就行：

1. conf：全局配置与外观资源（主题、字体、图标、语言等）
2. data/.siyuan/：忽略规则与内部状态标记（如搜索/索引忽略文件）
3. data/storage/：本地运行状态与配置缓存
4. data/plugins/：已安装插件本体及其配置。
5. 写味设置：sugar，顶栏合并→标题点标四个全点
6. 关闭云同步、关闭GitHub同步

### 链接方案：本质上就是思源自带的导出为markdown

#### 格式取舍

对勾的格式：可以用

不对勾的：少用

- [X] 有序和无序列表
- [X] 加粗、斜体
- [ ] 下划线
- [ ] 代码块
- [X] 引述
- [X] 1~6级标题
- [ ] ‍

--列表内换行时，尽量shift+enter，而非enter+backspace

#### 导出设置

两个设置综合的结果：思源导出设置+插件设置

###### 思源导出设置：

1. 引用：脚注+哈希值  
    锚文本块链，其它软件识别不了
2. 锚文本包裹符号：[[   ]]

‍

###### 插件设置：这个最关键

关键就在于**同步范围**

- 如果选用「数据目录（data目录）」，则会多出很多文件夹。
- 如果选用「笔记文件」，则会丧失链接功能——变成“右上角脚注+结尾脚注块全下载下来”

  > [[优化日志]]^n  
  > ......  
  > ......
  >
  > [^n]: # 优化日志  
  > <优化日志的全文>
  >

‍

### 尽量减少其它链接（内部链接、附件等）

相对路径非常敏感，很容易出问题

- 大范围剪切粘贴，相对路径必定完蛋，找不到链接指向的文件了
- 不停重构系统，相对路径全乱套  
  ——除了内容地图统筹，其它全都别用链接
